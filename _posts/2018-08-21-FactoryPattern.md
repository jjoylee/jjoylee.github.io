### Factory Pattern

#### Purpose(사용 목적)
sub class -> create object (Encapsulating Object creation)
클래스의 인스턴트 생성을 서브 클래스에서 결정
1. 객체 생성을 캡슐화한다.
2. 구상 클래스에 대한 의존성을 줄인다. 

#### Classification(분류)
1. Factory Method Pattern
2. Abstract Factory Pattern
